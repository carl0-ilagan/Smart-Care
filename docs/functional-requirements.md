## Functional Requirements

These requirements summarize the expected capabilities of the SmartCare telehealth platform for capstone documentation.

| # | Feature | Description |
| --- | --- | --- |
| FR-01 | User Registration | Patients, doctors, and admins can register/onboard with unique credentials. |
| FR-02 | User Authentication | Secure login, session handling, and role-based access per user type. |
| FR-03 | Appointment Scheduling | Patients can book/edit/cancel online or in-person visits; doctors manage availability. |
| FR-04 | Notifications | Send confirmations, reminders, and updates for appointments, prescriptions, and messages. |
| FR-05 | Messaging | Secure text/chat for follow-ups and clarifications between patients and providers. |
| FR-06 | Video Conferencing | Real-time video and audio between patients and doctors. |
| FR-07 | Document Upload/View/Share | Upload and view medical documents; patients can share specific records with providers securely. |
| FR-08 | Prescription Creation | Doctors draft e-prescriptions within the system. |
| FR-09 | Digital Signature | Doctors sign prescriptions electronically. |
| FR-10 | Prescription Delivery | System delivers signed e-prescriptions to patients. |
| FR-11 | Prescription History | Patients and doctors can view prior e-prescriptions. |
| FR-12 | Admin Approvals | Admins approve or reject patient/doctor accounts. |
| FR-13 | Admin Monitoring/Analytics | Admins can review system/activity metrics and logs. |
| FR-14 | Feedback Management | Capture and review feedback from users. |
| FR-15 | Backups | Admins can trigger daily JSON backups of backend data. |
| FR-16 | PWA Support | Installable web app with manifest/offline basics for improved accessibility. |
| FR-17 | Security | Enforce encrypted transport, role-based access, and audit logging for key actions. |
